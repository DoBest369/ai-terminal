# Termind 产品定位 · 智能 SSH 服务器运维工作台

> 一句话：**以 SSH 为入口、以 AI 为助手、以服务器资产管理与自动化运维为核心的智能服务器工作台。**
> 不是「更好看的 SSH 终端」，也不是「带 AI 聊天框的命令行」，而是让 SSH 工具具备
> 「理解环境 → 规划操作 → 评估风险 → 执行确认 → 结果验证 → 失败回滚」的完整智能运维闭环。

## 定位

- **不是**本地终端程序（不对标 iTerm / Warp / Windows Terminal / Tabby）。
- **是**智能 SSH 客户端 + 服务器管理工作台 + AI 运维助手，对标并升级
  **Xshell / FinalShell / MobaXterm / Termius / SecureCRT / WindTerm / 云平台 Web SSH**。
- 目标用户：个人开发者 / 站长 / VPS 用户 / 企业运维 / 云服务商。

## 三层产品形态

1. **专业 SSH 客户端**（基础，须稳）：SSH/SFTP/多标签/分组/密钥/端口转发/跳板/会话恢复。
2. **AI 智能运维助手**（差异化核心）：自然语言操作、命令解释、报错分析、日志总结、
   故障排查、部署辅助、安全提示——**结合当前服务器真实环境**给方案。
3. **服务器管理工作台**（平台化）：资产/状态/监控、批量操作、团队权限、审计、
   安全扫描、自动化脚本、部署模板、应用市场、云平台集成。

## 核心护城河

`AI + 当前服务器真实状态 + 安全执行机制 + 可恢复操作链路`

关键词：环境感知 · 安全确认 · 可回滚 · 可视化 · 自动化 · 多服务器 · 团队协作 · 云平台集成 · AI 排障 · 知识沉淀。

## 用户核心痛点（要解决的）

不会命令 · 不懂报错 · 不会部署 · 不会排查 · 害怕误操作 · 多服务器难管理 ·
服务器状态不可视 · 安全风险难发现 · 云平台与 SSH 割裂 · 运维经验无法沉淀。

## MVP 范围（先验证「用户愿不愿用 AI 辅助 SSH 工具」）

**必做**：SSH 连接 · 服务器分组 · SFTP · 多标签 · 密钥管理 · **AI 命令解释** ·
**AI 报错分析** · **常见排障工作流** · 服务器基础状态面板 · 危险命令提醒 ·
操作前确认 · 常用脚本模板。

**重点场景**：网站打不开排查 · Nginx 状态检查 · Docker 日志查看 · 磁盘清理分析 ·
SSL 证书检查 · Node.js 部署 · 静态网站部署 · 服务器初始化。

**暂缓**：完整团队权限 · 插件市场 · 私有化 · 复杂 CI/CD · 大规模批量运维 · 终端录制 · 审批流。

> **MVP 的差异化核心（AI 命令解释/报错分析/排障/环境感知/回滚/风险分级/模板）已全部双端落地**；并超出 MVP 实现了 **阶段 N 批量运维 + 主动巡检**。剩余为团队/云平台/生态化（路线后段）+ 多端铺开 + 真机打包发布。

## 迭代路线

1. **智能 SSH 客户端**：连接稳、SFTP 好用、AI 能解释命令和报错、常见问题能排查、安全确认完善。
2. **服务器工作台**：状态面板 · 日志中心 · 部署模板 · 一键初始化 · 操作回滚 · 服务器知识卡片。
3. **团队与云平台**：团队权限 · 操作审计 · 共享会话 · Web SSH · 云平台集成 · 工单联动。
4. **生态化**：插件系统 · 应用市场 · 自动化编排 · 私有化 · 企业知识库 · 多云管理。
